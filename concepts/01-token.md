# 01 · Token（词元）

> **第一梯队** | 缺了它，LLM 不存在

## 标准定义

Token 是 LLM 处理文本的最小单位。输入文本先被切分成 Token，模型以 Token 为粒度进行理解和生成。中文 1 字≈1.5 Token，英文 1 词≈1.3 Token。计费、上下文窗口、生成速度——全部以 Token 为单位。

## 费曼精炼（你的版本）

> **Token 是大模型感知、计算、预测世界的最小单位。**

LLM 世界唯一的原子——输入用它、思考用它、输出用它。

## 版本演进

| 版本 | 定义 | 问题 |
|------|------|------|
| v1 | Token ≈ 手机流量 | 只抓到「计费」，没抓到「意义」 |
| v2 | Token = 模型观察宇宙的像素 | 抓到「感知」，没抓到「计算」和「预测」 |
| **v3** ✅ | Token = 感知 + 计算 + 预测的最小单位 | 完整覆盖输入→处理→输出全生命周期 |

## 关键洞察

- 不是「容器」（如流量），Token **就是内容本身**
- 不是「字」（如汉字），Token 可能比字更细（「ChatGPT」→「Chat」「G」「PT」）
- Token 是模型唯一的世界观——它不知道「词语」「句子」「段落」，它只知道 Token
- 不同语言 Token 效率不同：同样意思，中文比英文少 ~40% Token

## 类比

像素之于屏幕 = Token 之于 LLM。像素是屏幕能显示的最小单元，Token 是模型能理解的最小单元。不同在于——像素只管显示，Token 管输入、处理、输出全部。
